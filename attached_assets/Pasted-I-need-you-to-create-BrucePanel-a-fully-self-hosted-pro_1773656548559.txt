I need you to create BrucePanel — a fully self-hosted, production-ready Node.js cloud hosting platform and PaaS (Platform as a Service) built entirely from scratch. Think of it as a self-hosted hybrid of Railway + Heroku + Predoctly + Netlify, with its own economy system, bot marketplace, admin CMS, and external developer API. Every feature described below must be fully implemented — no stubs, no placeholders.

STACK
Backend: Node.js (ESM), Express.js, PostgreSQL (pg Pool), JWT (jsonwebtoken), bcrypt (bcryptjs), dotenv, cors, multer, adm-zip, child_process spawn, SSE (Server-Sent Events), uuid, node-cron, node-pty (PTY terminal), systeminformation, ws (WebSockets for terminal), express-rate-limit, helmet, morgan, nodemailer (optional email)

Frontend: React 18, React Router v6, Vite, Tailwind CSS, Lucide React icons, xterm.js (terminal), Monaco Editor (code editor), Recharts (analytics charts), localStorage for JWT

Server: Single Express server. Serves built React from client/dist/. API prefix /brucepanel/. External API prefix /bp-api/v1/.

CREATE TABLE IF NOT EXISTS users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  avatar_url TEXT,

function startProcess(project) {
  const proc = spawn('sh', ['-c', project.start_command], {
    cwd: path.join(PROJECTS_DIR, project.id),
    env: { ...process.env, ...project.env_vars, PORT: project.port },
    stdio: ['pipe', 'pipe', 'pipe']
  });
  proc.stdout.on('data', d => broadcastLog(project.id, d.toString()));
  proc.stderr.on('data', d => broadcastLog(project.id, d.toString()));
  proc.on('exit', (code) => {
    if (project.auto_restart && code !== 0) setTimeout(() => startProcess(project), 3000);
  });
  processes.set(project.id, proc);
}

  });

  ws.on('close', () => pty.kill());
}

ZIP / FOLDER DEPLOY (Full Detail)
1. User hits "Upload" button → selects .zip or folder
2. POST /projects/:id/upload  multipart/form-data
3. Server: saves to bp_uploads/{id}-{ts}.zip
4. Extracts adm-zip → bp_projects/{id}/  (clears old files first, keeps .env)
5. Auto-detects runtime:
   - package.json → Node.js → npm install (or bun install if bun.lockb)
   - requirements.txt → Python → pip install
   - go.mod → Go → go build
   - Gemfile → Ruby → bundle install
6. Copies .env.example → .env (if no .env exists)
7. Runs build_command if set
8. Starts project with start_command
9. Assigns port, sets live_url
10. Returns { success, liveUrl, logsUrl }
11. Shows build SSE stream while installing
